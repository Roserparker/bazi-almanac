# 观象手记 · 云端每日 routine 说明书

> 目标：每天为「观象台 · BTC」写一段 90–160 字的 AI 观象评语（超出规则文案的新鲜话），
> 写入 `data/observatory-notes.json` 并 push——GitHub Pages 自动重发，页面即见。
> **手记缺席不影响页面**（前端优雅降级），所以 routine 挂了也无妨，补跑即可。

## 每次运行的步骤

1. 仓库 `Roserparker/bazi-almanac`（main）已在工作目录；先 `git config user.name '观星官' && git config user.email 'noreply@anthropic.com'`（云端无默认身份，缺此 commit 会失败）。
2. `npm install --omit=dev --no-audit --no-fund`（只需 lunar-javascript，不装测试用 devDeps）。
3. 取景：`TODAY=$(date -u +%F)`；`node scripts/observatory-note.mjs context $TODAY` → 当日观象全景 JSON
   （化机指数五因子、七日走向与周评、六爻本变卦与断财依据、奇门局与读盘、三法合参）。
4. **只依据这份 JSON 写评语**——不查行情、不编造价格与新闻（v1 立场：纯玄学观象；将来若要引入现实行情，需另行明确并过事实核查）。
5. 落笔：评语存工作目录 `._note.txt`，`node scripts/observatory-note.mjs write $TODAY ._note.txt && rm ._note.txt`（同日重跑会覆盖当日条目，天然幂等）。
6. 自检：`node scripts/observatory-note.mjs check` 必须 NOTES OK；`git status --short` 除 data/observatory-notes.json 外不应有其它改动。
7. `git add data/observatory-notes.json && git commit -m "note: 观象手记 $TODAY" && git push origin HEAD:main`（用 HEAD:main，兼容 detached HEAD 检出）。

## 评语风格铁律（与站规一致）

- **素静文人风**，90–160 字，一段成文；可用一句「观星官/臣观」式收束。
- **去恐吓化**：说「扬/敛/涩/持」，不说「凶/暴跌/必涨」；**永不给买卖指令**（页面已有总免责，评语内不必重复免责句）。
- 结构参考：先景（今日三法各见什么）→ 后断（重叠处/参差处的一句判读）→ 守则（一句心态或节奏建议）。
- 三法相左时，明说参差——「参差本是常态，重叠才是消息」是本站立场。
- 引用模型数字（指数、峰谷日）可以；引用外部世界（价格、新闻）不可以。

## 示例（2026-07-06 创刊条）

> 七月初六，丙午年甲午月叠火，己土得印而生——层运三扬，是本周的底色。惟流日辛巳带金气小逆，六爻鼎之旅财爻受克，奇门生门临己而戊落景宫：三尺参差，扬中有涩。观其重叠处：势不逆、而流转费力——宜持不宜追，把劲留给周六丙戌的高点去验证。观星官谨记：参差本是常态，重叠才是消息。
